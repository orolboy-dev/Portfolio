Git Tarixi

Dasturlashda kodlar nazoratini o'zgartirgan tizim

Git — bu dunyodagi eng ommabop taqsimlangan versiyalarni boshqarish tizimi (Distributed Version Control System) bo'lib, u 2005-yilda mashhur Linux operatsion tizimi asoschisi Linus Torvalds tomonidan yaratilgan. Loyiha Linux yadrosi (kernel) ustida ishlayotgan minglab dasturchilarning kodlarini tartibga solish va xavfsiz birlashtirish ehtiyojidan tug'ilgan.

Git-gacha bo'lgan tizimlar (masalan, SVN) markazlashgan edi va ishlash uchun doimiy internet talab qilardi. Git esa taqsimlangan (distributed) arxitekturani olib kirdi: har bir dasturchi o'z kompyuterida loyihaning barcha o'zgarishlar tarixini to'liq saqlab qoladi. Bu esa internet bo'lmaganda ham kodlar bilan bemalol ishlash va har qanday eski versiyaga osongina qaytish imkonini beradi.

Git-ning eng inqilobiy jihati — uning Branching (Tarmoqlanish) tizimidir. Dasturchilar asosiy kodga shikast yetkazmagan holda, alohida tarmoq ochib yangi funksiyalar qo'shishlari va barcha sinovlardan muvaffaqiyatli o'tgandan so'ng, ularni asosiy kodga (main/master) xavfsiz birlashtirishlari (merge) mumkin.

Bugungi kunda Git dasturiy ta'minot yaratish sohasining ajralmas qismidir. Kichik startaplardan tortib, Google, Microsoft, Facebook kabi gigant kompaniyalargacha jamoaviy ishlash, kodlar xavfsizligi va loyiha xronologiyasini yuritishda aynan Git tizimiga tayanadi. U dasturchiga o'z xatolarini to'g'irlash va jamoada chalkashliklarsiz ishlash erkinligini beradi.
